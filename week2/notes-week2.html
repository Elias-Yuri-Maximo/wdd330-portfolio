Notes 1 - programming Basics

No need to use semicolon, but it is a best practice to use them
Block: statements inside curly braces
Reserved words: abstract, await, boolean, break, byte, case, catch, char, class, const, continue, debugger, default, delete, do, double, else, enum, export, extends, false, final, finally, float, for, function, goto, if, implements, import, in instanceof, int, interface, let, long, native, new, null, package, private, protected, public, return, short, static, super, switch, synchronized, this, throw, throws, transient, true, try, typeof, var, volatile, void, while, with, yield
Data Types: String, Symbol (The symbol primitive data type was only introduced in ES6.) Number, Boolean, Undefined, Null
Everything else is considered an object.

Const: used when the value will not be reassigned. (using const on an object such as an array, will alow you to mutate(change the object))
let: used when the value can be reassigned
(Try to use const as much as possible in order to make the program predictable)
const and let are block scoped (they values only exist inside the blocks they are in )
Train to keep Global Variables to a minimum
